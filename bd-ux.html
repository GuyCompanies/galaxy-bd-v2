Why Platform is first-class now

Tracking, postback formats, link macros, and reconciliation rules all vary by Platform. Treating it as a string field caused config drift. As an entity it owns its own status, supports +Add inline, and cascades INACTIVE safely.

Cascade-inactive example

Cellxpert shows CASCADE-INACTIVE because its parent Network System "Old EU Network" was flipped INACTIVE. Reactivating the Network System auto-restores Cellxpert. Soft flag — no row deletion.

T1.6

INACTIVE confirmation pattern

Reused across all 4 sub-pages · the cascade safety net Tab 1
⚠️ Confirm INACTIVE — SuperBet

⚠️ Flip SuperBet to INACTIVE?

This will cascade-inactivate the following children:

Impact preview
  • 📦 14 Deals → all flip to CASCADE-INACTIVE
  • 📢 6 live Campaigns → auto-pause within 60s
  • 👥 23 CM allocations → freeze (no new traffic)
  • 🔗 3 Program links → marked INACTIVE
  • 📨 4 Affiliate Managers → email notification

The 4 safety rails

  • Impact preview before commit — never a blind flip.
  • Type-to-confirm — muscle-memory click can't fire it.
  • Soft flag, not hard cascade — children's effectiveStatus recomputes, no row deletion.
  • One-click reversible — flip parent back, children come back instantly.
🔁
Same modal, 4 entities: the pattern is identical for Network System, Program, Platform, Brand, and Account. One component, reused. Pnina learns it once.
📨
Audit + notification baked in: every flip writes a row to the audit log (who, when, why) AND fires an email to impacted Affiliate Managers. Compliance review-ready by default.
